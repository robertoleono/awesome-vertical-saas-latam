# Contributing

Thanks for helping make this list better.

## Rules

Each entry must:

1. Be open source (OSI-approved license) OR source-available with explicit terms.
2. Be relevant to LatAm SMB verticals (dental, salons, law, accounting, restaurants, real estate, gyms, etc).
3. Be actively maintained (last commit within 12 months).
4. Use the format: `- [Name](URL) - License - One-line description in present tense.`
5. State the license inline when it is restrictive: AGPL, SSPL, BSL, CC-BY-NC, Fair-code.

## Categories

Add a new category only when 3+ entries justify it.

## PR format

- One PR per category change.
- PR title: `Add [project] to [category]`.
- PR body: brief reason + link to project.

## Closed-source tools

Reference closed/commercial tools only as "alternative-to" context for OSS entries, never as standalone entries.

## Code of conduct

Be respectful. Keep technical discussions on-topic.
